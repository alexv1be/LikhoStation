# План разработки проекта: Станция Лихо: Изнанка (Неделя 1)

**Основная цель недели:** Создание технического каркаса приложения и реализация базового прототипа перемещения в условиях ограниченной видимости («Серой Хмари»).

---

## 🏗 Ключевые задачи на неделю

### 1. Архитектура игрового движка (WinForms)
* **Инициализация графического ядра:** Настройка основной формы, активация режима `DoubleBuffered` для плавной отрисовки без мерцания.
* **Игровой цикл (Game Loop):** Реализация системы обновления кадров на базе `Timer`. Разделение логики на два этапа: `Update` (расчет координат, состояний) и `Draw` (отрисовка через GDI+).

### 2. Система управления и физика игрока
* **Обработка ввода:** Настройка отзывчивого управления (WASD/стрелки). Реализация системы, позволяющей обрабатывать нажатие нескольких клавиш одновременно (для движения по диагонали).
* **Кинематика:** Настройка скорости персонажа и реализация системы коллизий (ограничение передвижения границами экрана/препятствиями).

### 3. Визуализация «Серой Хмари» (Система видимости)
* **Динамический маскинг:** Реализация графического слоя, перекрывающего обзор. Использование `GraphicsPath` для отрисовки «окна видимости» вокруг игрока.
* **Связка координат:** Программирование зависимости центра области видимости от текущих координат игрока в реальном времени.

### 4. Прототип взаимодействия
* **Триггер взаимодействия:** Базовая логика обработки клавиши `E` при нахождении игрока рядом с объектом (подготовка базы для подбора предметов и диалогов).

---

## 🎯 Ожидаемый результат к концу недели
Работающее Windows-приложение с «черным экраном», по которому перемещается маркер игрока, «разрезая» туман вокруг себя. Это станет доказательством работоспособности выбранной технологии (GDI+) для реализации уникальной атмосферы игры.
